# 后续内容规划：Season 3 及以后

> 原则：后续 Season 先定方向，不提前凑篇数。每一季都必须围绕真实问题、真实产品或可动画化 Demo 展开。

---

## 一、总主线

整个 AI Native 项目围绕一条主线：

> **AI 如何把软件从“功能系统”，推向“目标执行系统”。**

不同 Season 负责不同层级：

```text
Season 1：为什么变
Season 2：产品怎么设计
Season 3：系统怎么实现
Season 4：真实产品做对了什么
Season 5：开发者和创业机会在哪里
```

---

## 二、Season 3：AI Native 技术架构

定位：

从产品设计进入工程实现。

Season 2 讲：

> 产品应该怎么设计。

Season 3 讲：

> 系统应该怎么搭出来。

核心要求：

- 不做概念百科；
- 不讲纯理论；
- 每篇必须有工程场景、架构图或伪代码流程；
- 尽量结合一个小型 Demo 系统。

候选问题：

### S3-01 AI Native 技术栈到底包括什么？

回答：

- LLM、Prompt、Memory、RAG、Tools、Workflow、Agent、MCP 分别处在哪一层；
- 一个 AI Native 系统的最小架构是什么。

动画 / 图示：

```text
用户目标
  ↓
产品层
  ↓
Agent / Workflow
  ↓
Memory / RAG
  ↓
Tools / APIs / MCP
  ↓
外部系统
```

### S3-02 Prompt 在系统里到底放哪一层？

回答：

- System Prompt、User Prompt、Tool Prompt、Workflow Prompt 的分工；
- 为什么 Prompt 不该散落在代码里；
- Prompt 如何从前台输入变成后台配置。

### S3-03 Memory 如何实现：短期、长期、项目上下文

回答：

- 短期上下文和长期记忆的区别；
- 用户记忆、项目记忆、组织记忆如何分层；
- 什么时候用数据库，什么时候用向量检索。

### S3-04 RAG 什么时候有用，什么时候是负担？

回答：

- RAG 解决什么问题；
- 哪些场景不该上 RAG；
- 检索、重排、引用、可信度如何设计。

### S3-05 Tool Calling / MCP 如何变成可控系统？

回答：

- 工具注册；
- 参数设计；
- 权限确认；
- 执行结果回传；
- 错误处理和回滚。

### S3-06 Agent Workflow 如何落地，而不是 Demo？

回答：

- Planner、Executor、Verifier 的分工；
- 什么时候需要 Agent，什么时候普通 Workflow 更好；
- 如何避免无限循环和幻觉执行。

### S3-07 AI Backend 和传统 Backend 有什么不同？

回答：

- 传统 API 后端和 AI 调度后端的区别；
- 状态、上下文、工具、任务队列、审计日志；
- 为什么未来后端会多一层 AI Orchestration。

### S3-08 如何评估 AI Native 产品质量？

回答：

- 正确率、完成率、成本、延迟；
- 用户信任；
- 工具调用成功率；
- 任务闭环率。

Season 3 不一定全部一次完成，可以根据 Season 2 反馈选择 4-6 个优先题。

## 三、Season 4：AI Native 案例拆解

定位：

拆真实产品，不讲抽象概念。

目标：

> 看懂优秀 AI Native 产品到底做对了什么。

候选案例：

### Cursor

重点：

- 为什么它不只是代码生成；
- 如何把 IDE、上下文、Agent、工具调用结合起来；
- 为什么程序员感受到的是工作方式变化。

### Perplexity

重点：

- 为什么它不像传统搜索；
- 搜索、阅读、引用、答案生成如何组合；
- 为什么它改变的是信息获取流程。

### ChatGPT

重点：

- 从聊天机器人到工作入口；
- Memory、Tools、Projects、GPTs、Agents 的组合；
- 它是否在变成新的操作层。

### Claude

重点：

- 为什么开发者喜欢 Claude；
- Artifacts / Projects / MCP 如何影响工作流；
- 它的产品形态和 ChatGPT 有什么不同。

### NotebookLM

重点：

- 文档理解产品为什么有价值；
- 它如何围绕资料构建上下文；
- 为什么它不是普通问答工具。

### Granola / Plaud

重点：

- 会议记录为什么不只是转写；
- AI 如何从音频变成行动项、决策和后续任务；
- 这类产品的边界在哪里。

### Manus / Devin / Browser Use

重点：

- Agent 执行任务的机会和风险；
- 浏览器操作、任务拆解、工具调用；
- 哪些是 Demo，哪些接近真实价值。

Season 4 每篇必须包含：

```text
产品解决什么真实问题
    ↓
它的旧世界替代方案是什么
    ↓
它的 AI Native 结构在哪里
    ↓
它做对了什么
    ↓
它还没解决什么
```

## 四、Season 5：开发者与创业机会

定位：

从产品和技术回到个人与创业。

核心问题：

> 独立开发者、程序员、小团队，如何在 AI Native 生态里找到自己的位置？

候选方向：

### S5-01 独立开发者还适合做完整 App 吗？

讨论：

- 完整产品的机会与难度；
- 垂直场景 vs 通用工具；
- 小团队如何避开大厂正面竞争。

### S5-02 什么能力适合做成 AI 可以调用的服务？

讨论：

- API、MCP Server、数据源、浏览器能力；
- 能力产品如何定价；
- 被 AI 调用的产品如何获得分发。

### S5-03 AI Native 时代的开发工具机会

讨论：

- Debug、Eval、Prompt 管理、Agent 监控；
- 开发者基础设施；
- 小工具如何变成生态组件。

### S5-04 AI Native 产品如何商业化？

讨论：

- 订阅制是否仍然成立；
- 按任务、按结果、按调用计费；
- 成本、模型费用和毛利。

### S5-05 普通程序员如何转向 AI Native Developer？

讨论：

- 从写功能到设计目标流程；
- 从页面思维到工具调用思维；
- 从单体 App 到生态位置。

## 六、后续规划原则

每个 Season 开始前都要重新回答：

1. 这一季解决什么真实问题？
2. 核心观众是谁？
3. 每篇有没有真实产品案例？
4. 没有案例时，能不能用动画 Demo 讲清楚？
5. 这一季和上一季有什么边界？
6. 是否真的需要这么多篇，还是可以合并？

如果回答不清楚，就不要写。
