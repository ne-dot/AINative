## 总主线

整个项目围绕一个核心判断：

> **AI Native 不是 AI 工具合集，而是软件从“功能系统”走向“目标执行系统”的范式迁移。**

换句话说：

- 过去的软件围绕功能；
- 未来的软件围绕目标；
- 过去用户寻找软件和功能；
- 未来用户表达目标，由 AI 调度能力、工具和服务。

## 内容生产原则

每一篇内容都先回答：

1. 观众为什么要看？
2. 它解决什么真实问题？
3. 有没有真实产品案例？
4. 没有案例时，能不能用动画 Demo 讲清楚？
5. 是否有一个可复述的结论？

如果没有真实问题、真实场景或可动画化结构，就先不写。
